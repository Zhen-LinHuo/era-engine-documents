# 数据文件

在 `FSM 模式` 下编写游戏的另一个重要部分是数据文件。无论是口上、角色数据、物品和装备数据，亦或是其他更多类型的信息，都需要存储为数据文件以供程序查询。

目前，ERA-Engine支持以下数据文件格式：

## 已经支持的格式

### CSV

CSV（逗号分隔值）是一种文本数据格式，它使用逗号分隔不同的值，并使用换行符分隔记录。CSV 数据以纯文本形式存储表格数据（数字和文本），其中每一行通常代表一条数据记录。

CSV 格式最适合表示记录集或记录序列，其中每条记录都具有相同的字段列表。这对应于关系数据库中的单个关系，或者对应于典型电子表格中的数据（但不包括计算结果）。

CSV 格式是保存数据最简单的格式，也是 Emuera 游戏保存数据的方式。

## 计划支持的格式

### INI

INI 文件是一种计算机软件的配置文件，它由纯文本构成，结构和语法以键值对的形式组织成不同的部分。

INI 文件易于阅读和解析，因此适用于不太复杂的配置文件。

Godot 原生支持 INI 格式的文件，因此我们很快也将支持这种数据文件管理方式。

### XML

XML（Extensible Markup Language, 可扩展标记语言）是一种用于存储、传输和重构数据的标记语言和文件格式。它定义了一套规则，用于以既人可读又机器可读的格式对文档进行编码。

XML 已广泛应用于互联网数据交换。数百种使用 XML 语法的文档格式已被开发出来，包括 RSS、Atom、Office Open XML、OpenDocument、SVG、COLLADA 和 XHTML。

与 CSV 相比，XML 可以提供更易读、结构化的数据组织方式。
