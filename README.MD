## How to Contribute to the Project

This project is constructed with `DocFX`, deployed on `GitHub Page`.

该项目使用 `DocFX` 编写，通过 `GitHub Page` 部署。

To make any update to this document project, please follow this guidance:

想要为本项目做出任何贡献和更新，请遵从以下指引：

### JetBrains IDE & DocFX

Since `ERA-Engine` is using `C#` and `GD Script`, we recommend using `JetBrains Rider`. It has community version which is free to use:

由于 `ERA-Engine` 使用 `C#` 和 `GD Script`，我们推荐使用 `JetBrains Rider`。它有社区版本，可以免费使用：

- [Install JetBrians Toolbox to Manage All Your JetBrains Products](https://www.jetbrains.com/toolbox-app/)
- [Directly Download From Official Website](https://www.jetbrains.com/rider/)

After installation of `JetBrains Rider`, you can install `DocFX` as follows:

安装完 `JetBrains Rider` 并用其打开本项目的存储库后，使用如下方式安装 `DocFX`：

1. Make Sure you installed `.Net SDK`
2. Open `Terminal` in `JetBrains Rider`
3. Type `dotnet tool update -g docfx`

### Editing

Once you opened this repository in Rider, you can find all document files in `docs` folder. all API references will be automatically generated by `DocFX` and stored in `api` folder.

当你用 `JetBrains Rider` 打开本项目的存储库后，你可以在 `docs` 文件夹中找到所有的文档文件。所有的 API 文档将会被 `DocFX` 自动生成并存储在 `api` 文件夹中。

Simply, you can use `Markdown` to construct your page, read [here](https://dotnet.github.io/docfx/docs/markdown.html) for official instructions.

你可以简单地使用 `Markdown` 来构建你的页面，阅读 [这里](https://dotnet.github.io/docfx/docs/markdown.html) 获取官方指引。

### Build & Commit

Check content in `docfx.json` file to make sure you correctly specified the location of `ERA-Engine` `csproj` file.

首先检查 `docfx.json` 文件，确保你正确地指定了 `ERA-Engine` `csproj` 文件的位置。

Run `docfx .\docfx.json --serve` to build the static website and open it in your browser.

运行 `docfx .\docfx.json --serve` 来构建静态网站并在浏览器中打开。

Once you make sure everything is good, commit and make a pull request to merge your commits. `GitHub Actions` will automatically deploy the website to `GitHub Page` based on the `_site` folder content in `main` branch.

当你确定网站正常构建且内容均没有问题后，发布 commit 并且发起一个 pull request 来合并你的 commit。`GitHub Actions` 将会基于 `main` 分支中的 `_site` 文件夹内容自动部署网站到 `GitHub Page`。
