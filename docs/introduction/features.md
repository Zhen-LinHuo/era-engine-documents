# 功能特性

ERA-Engine 在继承经典 ERA 游戏开发理念的基础上，引入了多项现代化设计。本章将逐一介绍引擎的核心功能特性。

---

## FSM 模式（推荐）

ERA-Engine 使用**有限状态机（FSM）模式**来组织游戏代码，充分利用 Godot 引擎的信号系统和节点能力。

```gdscript
# FSM 模式示例
extends ERAState

func on_enter():
    TXT("欢迎来到冒险者公会。")
    BTN("接受任务", "quest_board")
    BTN("招募同伴", "recruit")
    BTN("离开", "town_square")
```

- ✅ 与 Godot 引擎深度集成，可充分利用场景、信号和动画功能
- ✅ 代码结构清晰，易于维护和扩展
- ✅ 支持复杂的分支逻辑和动态 UI 构建
- ✅ 更适合中大型项目

> 💡 **建议**：如果你是新项目，推荐从 FSM 模式开始。它提供了更好的可扩展性和现代开发体验，是 ERA-Engine 的设计重点。

## 数据驱动架构

ERA-Engine 强调**数据与逻辑分离**。游戏的配置、角色属性、物品定义、对话脚本等静态数据，全部通过结构化文件管理，而非硬编码在脚本中。

### 支持的数据格式

| 格式 | 适用场景                         | 优势                       |
| ---- | -------------------------------- | -------------------------- |
| CSV  | 表格数据（角色属性、物品清单等） | 可用电子表格编辑，便于协作 |
| JSON | 结构化配置（系统设置、嵌套数据） | 灵活，广泛支持             |
| 其它 | 按需扩展                         | 开放的数据文件格式体系     |

### 数据驱动的优势

- **非程序员友好**：写作者和数值策划可以直接在 CSV 或电子表格中编辑游戏数据，无需接触代码
- **热重载**：修改数据文件后可即时在游戏中看到效果，大幅提升迭代速度
- **易于模组化**：模组开发者只需提供新的数据文件，即可添加角色、物品或剧情
- **版本控制友好**：纯文本格式便于 Git 等工具追踪变更历史

## 双语言架构

ERA-Engine 采用 C# 与 GDScript 协同工作的双语言设计（详见 [Godot 与脚本语言](godot-and-languages.md)）：

- **GDScript 层** — 游戏逻辑 / 状态脚本 / 对话内容
- **C# 核心层** — FSM 引擎 / MVC 框架 / 数据管理
- **Godot 引擎** — 渲染 / 输入 / 音频 / 跨平台

- **C# 核心**提供稳定、高性能的基础设施
- **GDScript 层**让创作者快速编写游戏内容，无需关心底层细节

## 模块化视图系统

ERA-Engine 的 UI 基于可组合的模块化视图组件构建。三个核心概念构成了一个灵活的 UI 体系：

### Unit（单元）
游戏中的基本角色实体，包含该角色的属性和行为定义。
```
Unit: "爱丽丝"
  - 体力: 100
  - 好感度: 45
  - 位置: 公会大厅
```

### Container（容器）
承载和分组 Unit 的逻辑容器。一个 Unit 可以同时属于多个 Container。
```
Container: "队伍"
  - 爱丽丝
  - 鲍勃
Container: "酒馆客人"
  - 爱丽丝
  - 查理
  - 黛安娜
```

### Suite（套件）
一组可复用的 UI 组件组合，定义了特定场景下的界面布局和交互模式。
```
Suite: "对话套件"
  - 对话文本区
  - 选项按钮组
  - 角色立绘区
  - 状态概览栏

Suite: "商店套件"
  - 物品列表
  - 价格/余额显示
  - 购买/出售按钮
```

### 组合的力量

通过组合不同的 Unit、Container 和 Suite，你可以快速构建丰富的游戏场景：

> 「爱丽丝」（Unit）属于「队伍」（Container），在「酒馆」场景中，使用「对话套件」（Suite）显示她的台词和互动选项。

## 功能特性总览

| 特性               | 说明                                           |
| ------------------ | ---------------------------------------------- |
| 数据驱动           | 结构化数据文件管理游戏数据，热重载支持          |
