# Godot 与脚本语言

## 什么是 Godot？

[Godot](https://godotengine.org/) 是一个开源、免费的游戏引擎，以其轻量级设计、强大的场景系统和直观的节点架构而闻名。它支持 2D 和 3D 游戏开发，并内置了完整的编辑器、动画系统、物理引擎和音频处理功能。

Godot 的核心概念：

- **节点（Node）**：一切皆节点。一个角色、一个按钮、一个音效，都是一个节点。节点是最基本的构建块。
- **场景（Scene）**：场景是节点的集合，以树形结构组织。一个场景可以是一个完整的关卡，也可以是一个可复用的角色或 UI 组件。
- **信号（Signal）**：Godot 内建的观察者模式实现。节点可以通过发送和接收信号来解耦通信，例如「按钮被按下」→「播放音效」。

## 为什么选择 Godot？

在众多游戏引擎中，我们选择 Godot 作为 ERA-Engine 的底层基础，基于以下考量：

| 优势              | 说明                                           |
| ----------------- | ---------------------------------------------- |
| **完全免费**      | MIT 许可证，无版权费，无分成                   |
| **开源透明**      | 源代码完全公开，可根据需要自定义引擎行为       |
| **轻量高效**      | 下载包仅约 50MB，启动迅速，适合迭代开发        |
| **原生 C# 支持**  | 通过 .NET 集成提供完整的 C# 开发体验           |
| **强大的编辑器**  | 内置场景编辑器、动画工具、调试器和性能分析器   |
| **跨平台部署**    | 一次开发，导出到 Windows、macOS、Linux、Android、iOS、Web |

相比 Unity（较重、收费政策变化频繁）和 Unreal（以 3D 高画质为重点），Godot 对于以 2D UI 为主的 ERA 类游戏来说，是更轻便、合适的选择。

## C# — 引擎核心语言

ERA-Engine 的核心引擎部分使用 **C#** 编写，负责处理底层逻辑和性能敏感的任务。

### 为什么核心用 C#？

- **性能**：C# 是编译型语言，执行效率远超 GDScript，适合处理复杂计算、数据管理和状态机调度等核心任务
- **类型安全**：静态类型系统在编译阶段就能捕获大量错误，保障引擎核心的稳定性和可维护性
- **.NET 生态**：可以充分利用 .NET 生态系统中的成熟库（如 JSON 解析、CSV 处理、网络通信等）
- **IDE 支持**：与 Visual Studio、Rider 等 IDE 深度集成，提供智能补全、重构和调试功能

### C# 在 ERA-Engine 中负责什么？

- 状态机（FSM）的调度与运行
- MVC 框架的核心实现（Controller 和部分 Model）
- 数据加载与解析（CSV、JSON、XML）
- 性能敏感的底层操作
- 跨平台兼容性处理

## GDScript — 游戏脚本语言

ERA-Engine 的**游戏逻辑和内容**部分使用 **GDScript** 编写。GDScript 是 Godot 的原生脚本语言，语法类似 Python，易于学习。

### 为什么游戏内容用 GDScript？

- **与 Godot 紧密集成**：GDScript 可以直接访问 Godot 的节点树、信号系统和场景资源，无需中间层
- **简单易学**：缩进式语法、动态类型，降低了非程序员（如剧本写作者）的入门门槛
- **快速迭代**：修改脚本后无需重新编译，可在编辑器中即时看到效果
- **热重载**：支持运行时的脚本热更新，极大提升开发效率

### 在 ERA-Engine 中，GDScript 开发者会看到什么？

开发游戏内容时，你主要接触的是 GDScript 编写的「状态」文件。一个典型的状态看起来像这样：

```gdscript
extends ERAState

func on_enter():
    TXT("你走进了一间昏暗的房间。")
    TXT("桌上放着一封信和一把钥匙。")
    BTN("阅读信件", "read_letter")
    BTN("拿起钥匙", "take_key")
    BTN("离开房间", "leave_room")
    BOX("status", "体力: " + str(player.stamina))
```

熟悉的 `TXT()`、`BTN()`、`BOX()` 函数让 ERA 老手能够快速上手，同时享受 GDScript 带来的完整编程能力。

## 何时使用哪种语言？

| 场景                          | 推荐语言    | 原因                           |
| ----------------------------- | ----------- | ------------------------------ |
| 编写游戏剧情和对话            | GDScript    | 快速迭代，语法简单             |
| 实现游戏逻辑（状态）          | GDScript    | 与 Godot 紧密集成，热重载      |
| 引擎核心架构                  | C#          | 性能要求高，需要类型安全       |
| 数据处理与解析                | C#          | 利用 .NET 生态，高效处理       |
| 自定义 UI 组件                | C# 或 GDScript | 根据复杂度选择              |
| 简单的一次性工具脚本          | GDScript    | 快速编写，无需编译             |

简单来说：**C# 筑地基，GDScript 盖房子**。核心引擎提供稳定、高性能的基础设施；游戏创作者在 GDScript 中自由地编写内容，无需关心底层细节。
